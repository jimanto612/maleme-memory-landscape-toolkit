# Maleme Memory Landscape Toolkit

**Open-source architecture thesis toolkit for memory landscapes, GIS/CAD site analysis, AI-assisted research, and design documentation.**

This repository is connected to an architecture thesis project at the **Technical University of Crete**. It explores the old **Maleme airfield in Crete** as a landscape of war, scar, memory, recovery, and peace. The project treats architecture not only as the production of form, but as a method for reading, documenting, and transforming places that carry historical weight.

The aim of this repository is to make the thesis process public, reusable, and critically useful for other architecture students, researchers, and educators working with memory landscapes, military remains, heritage sites, and landscape-architecture interventions.

---

## Core Idea

The thesis is structured around the conceptual sequence:

```text
WAR → SCAR → MEMORY → RECOVERY → PEACE
```

This sequence is used as a design-research framework for reading the site, organizing historical material, producing maps and diagrams, and developing architectural interventions that respond to memory without turning trauma into spectacle.

The project asks:

- How can architecture act as a tool of remembrance without becoming symbolic decoration?
- How can a damaged or historically charged landscape be documented with precision and transformed with restraint?
- How can GIS, CAD, drawing, writing, and AI-assisted research become part of one transparent architectural workflow?
- How can thesis methodology become useful beyond a single academic submission?

## Project Scope

The toolkit focuses on four connected layers of architectural research:

### 1. Site and Landscape Analysis

Methods for reading the physical, historical, and atmospheric qualities of a place. This includes terrain, access, traces, voids, military remains, vegetation, exposure, horizon, and the relationship between memory and spatial experience.

### 2. Mapping and Documentation

A structured workflow for using tools such as QGIS, CAD, and drawing software to produce maps, analytical drawings, diagrams, and presentation boards with consistent architectural logic.

### 3. Theoretical and Historical Synthesis

A method for connecting historical research, architectural theory, phenomenology, memory studies, and design intention without reducing the project to generic symbolism.

### 4. Design-Research Translation

A workflow for translating research into architectural interventions, material studies, spatial sequences, and final boards. The focus is on clarity, restraint, atmosphere, and structural logic.

## Why This Matters

Architecture students often produce complex thesis projects, but the actual workflow usually remains hidden: scattered notes, unstructured prompts, isolated drawings, and final boards with little explanation of how the work was developed.

This project tries to make that process visible.

It proposes that architectural research should be:

- **transparent**, so others can understand how conclusions were formed;
- **reusable**, so future students can adapt the workflow;
- **critical**, so AI is used as a tool for thinking, not as a machine for producing empty text;
- **spatial**, so research remains connected to matter, ground, light, movement, and atmosphere;
- **ethically restrained**, especially when working with sites of trauma and memory.

## Methodological Position

This project does not treat AI as a replacement for architectural judgment.

AI is used here as a support system for:

- organizing research;
- testing arguments;
- comparing precedents;
- improving writing clarity;
- building reusable documentation;
- creating critical prompts;
- exposing weak conceptual logic before it reaches the final design stage.

The architectural decisions remain grounded in site, material, atmosphere, structure, proportion, and human experience.

## How to Use This Toolkit

You can use this repository as a reference structure for your own architecture thesis or research project.

A suggested workflow:

1. Start with the **site context** and define what kind of landscape you are studying.
2. Build a **mapping system** using GIS, CAD, and diagrammatic layers.
3. Write a short **theoretical framework** that explains the project’s intellectual position.
4. Translate research into a **spatial sequence**, not just a concept statement.
5. Use the templates to produce clear boards, captions, diagrams, and written documentation.
6. Adapt the AI prompts carefully, always checking the output against architectural judgment and real sources.

---

## Ethical Note

The Maleme airfield is treated as a historically charged landscape. Any architectural intervention in such a place must avoid superficial dramatization. The goal is not to aestheticize trauma, but to create a disciplined spatial framework through which memory, absence, material trace, and landscape can be read with care.
